**日期：** 2026年3月14日

## 1. 软件功能概述

本软件是一个集成物联网（IoT）监控与数据分析的 Web 平台，旨在为用户提供实时、直观的设备管理与数据可视化服务。

### 核心功能模块：
*   **物联网调试面板 (`HoireDebug`)：**
    *   与 Hoire 物联网平台进行交互的核心组件。
    *   支持获取物联网设备 Token，查询气象站、虫情设备、摄像头等设备列表。
    *   实现设备订阅逻辑，将前端请求转发至后端代理。
*   **地理空间监控 (`MapSection`)：**
    *   在地图界面上可视化展示所有已连接物联网设备的位置与状态。
*   **时间线分析 (`TimelineSection`)：**
    *   记录并展示设备数据的历史趋势、告警事件及操作记录。
*   **实时监控 (`MonitoringSection`)：**
    *   提供设备运行数据的实时仪表盘，显示关键指标。
*   **天气组件 (`WeatherWidget`)：**
    *   展示当前环境气象信息，辅助设备数据分析。
*   **管理后台 (`AdminPage`)：**
    *   提供系统配置、用户管理及高级调试功能。
*   **通用组件：**
    *   `Header`：全局导航与状态栏。
    *   `JoinUsButton`：引导用户参与或反馈的交互入口。

## 2. 系统架构

本软件采用典型的全栈架构，确保前端交互的流畅性与后端通信的安全性。

### 架构组成：
*   **前端 (Frontend)：**
    *   **技术栈：** React 18+, TypeScript, Tailwind CSS, Vite。
    *   **职责：** 负责 UI 渲染、状态管理、用户交互及调用后端 API。
*   **后端 (Backend)：**
    *   **技术栈：** Express.js (Node.js)。
    *   **职责：**
        *   **安全代理：** 代理前端对 Hoire 平台的 API 请求，隐藏敏感的 API Token，防止在前端暴露。
        *   **跨域处理：** 解决前端直接请求 Hoire 平台可能存在的跨域问题。
        *   **业务逻辑：** 处理订阅流程、数据转换及错误日志记录。
*   **数据流向：**
    *   前端 -> `/api/hoire/*` (Express 后端) -> Hoire IoT 平台 API。

## 4. 当前卡点 (Blockers)

目前项目处于集成调试阶段，以下卡点直接影响功能的完整交付：

### 4.1 技术逻辑卡点 (API 响应处理)
*   **问题描述：** 在执行设备订阅流程时，Hoire 平台返回 `1006` 状态码，明确提示“订阅已存在，无须重复订阅”。
*   **当前影响：** 前端代码逻辑将非 `200` 的响应均视为异常并抛出错误，导致订阅流程在遇到“已订阅”状态时中断，无法完成后续所有设备的订阅请求。

### 4.2 环境与调试卡点 (浏览器缓存)
*   **问题描述：** 浏览器对 JavaScript 代码文件（特别是 `HoireDebug.tsx` 编译后的产物）存在强缓存机制。
*   **当前影响：** 后端及前端代码逻辑的修改在服务器端已生效，但浏览器端无法及时加载最新代码，导致调试过程陷入“修改-刷新-无效”的循环，无法验证修复方案的有效性。
